Summary

The article redefines software agents as self-writing programs that use external LLMs to determine their actions, emphasizing that agents behave like distributed systems and can fail in complex ways when interacting with real-world infrastructure. It explores various agent failure modes and argues that making agents fault-tolerant requires adopting distributed systems and database techniques, such as transactional logs, to ensure reliability. The author introduces LogAct, a system that uses shared logs for safer, auditable, and recoverable agent actions.
